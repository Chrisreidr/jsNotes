Data Stuctures

Sources of Data

  1. From the program itself: Data written directly in the source code. Such as status messages that will be displayed on the webpage based on user actions.
  2. From the UI: Data input from the user interface (webpage) or data written in the DOM
  3. From external sources: Data fetched for example from a web API (Apllication Programming Interface). Used to get data from other web applications. Usually in the form of JSON, which is esstntially a long sting of text that can be converted to Javascript objects because of the same formatting as Javascript objects and arrays.

There are usually collections of data (conversion rates, movie, recipes, numbers, etc.) that need to be stored somewhere. We store them in one of the four built-in data structures.

When to use each Data structure

    Arrays and Sets

    Simple list of values. Simply contain values without any description.

  • Array: When you need ordered list of values (might contain duplicates) or when the data needs to be manipulated.
  • Set: Use with unique values when high-performance (10 times faster searching for or deleting an item than arrays) is necessary and to remove duplicates from arrays.

    • Objects and Maps

    Key value pairs. Keys are used to describing each value

  • Object: Used before ES6. Easier to write and access values via the dot (.) and bracket ([]) notations. Use when you need functions (methods) as values are required. Also allows us to use the .this keyword to access properties of the same object.
    Always use function expression as methods in objects and NOT arrow functions because it cannot have its own this keyword and will returned undefined. But it can inheret the this keyword from its parent scope.
  • Map: Offer better performance, keys can have any data type, are easy to iterate and compute the size of. Use when you simply need to map values and keys that are not strings

Sets

Set objects are collections of values. You can iterate through the elements of a set in insertion order. A value in the Set may only occur once; it is unique in the Set's collection.

Maps

Maps are data structures used to map values to keys. Like in objects, data is stored in the key value pairs in maps. In objects keys are bascially always strings. While maps can have any type of key including arrays, objects, or other maps.

  • const x = new Map() - Begin by creating an empty map, then simply add the key and value via the set method. It returns the value immediately, meaning you can chain more set method to add multiple keys and values at once.
  • x.set () - Begin by creating an empty map, then simply add the key and value via the set method. It returns the value immediately, meaning you can chain more set method to add multiple keys and values at once.
  • x.get () - We are able to revrieve any value and property from the maps by passing in the name of the key
  • x.delete () - Will delete the key and value from a map of the key specified in the paranthese
  • x.size () - Returns the number of elements in a map object
  • x.clear () - Removes all elements from a map
  • x.values () - We are able to revrieve all values from the maps by leaving the paranthese empty

Maps: Iteration

We can also create a new map already populated with values by creating an array containing individual arrays with a key and value pair. Which can also be accomplished by calling Object.entries() on an object.

In order to convert an object to a map, type const objectToMap = new Map(Object.entries(objectProperty));
